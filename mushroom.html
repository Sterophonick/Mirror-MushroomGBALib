Overview

Mushroom is a simple C library for the Nintendo Game Boy Advance. It provides a thin layer over the GBA hardware to allow for maximum control and flexibility in development.

Mushroom GBALib is public domain. See below for details.

Requirements

Most of the software listed here can be found at http://www.gbadev.org.

Getting Started

Extract the Mushroom package to a new directory on your computer (you have probably already done this).

Edit <mushroom_dir>/build/Makefile_defines
Change the following variables to match your system:
GCC_VERSION, GCC_DIR, EMU

Enter the directory <mushroom_dir>/samples/newproject/src
Type "make"
The project should compile.

Then type "make run"
This should run the GBA ROM in the emulator specified in Makefile_defines.

You will also have to make sure you have gfx2gba installed and specified in Makefile_defines (the GFX2GBA variable). This is because the other samples require gfx2gba to convert bitmaps to C code.

Creating A New Project

See Getting Started first to make sure you can compile ok.

Create a directory anywhere on your computer with the name of your project.

Copy the contents of <mushroom_dir>/samples/newproject into your project directory.

Rename the new.c file in <project_dir>/src to the same name as your project directory.

Edit <project_dir>/src/Makefile and change PROJECT_NAME to the name of your project directory.

Enter the directory <project_dir>/src
Type "make"
The project should compile.

Then type "make run"
This should run the GBA ROM in the emulator specified in Makefile_defines.

Edit your source code and compile again (repeat indefinately).

Documentation

Mushroom API Reference
This lists all the functions in Mushroom. However, it doesn't list all the macros. Check the Mushroom header files for macros if you need something more low level.

This is probably the best freely available reference for the GBA. Mushroom takes most of it's naming conventions from this reference.
